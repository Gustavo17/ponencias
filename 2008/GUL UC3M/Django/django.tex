\section{El Framework}
\subsection{MVC RoR vs MVC Django}
\begin{frame}{MVC RoR}
  \begin{itemize}
    \item Modelo: Abstracción del acceso a datos.
    \item Vista: Presentación de los datos al usuario.
    \item Controlador: Lógica de programa.
    \item Rutas: Redirige las consultas al controlador apropiado.
  \end{itemize}  
\end{frame}

\begin{frame}{MVC Django}
  \begin{itemize}
    \item Modelo: Abstracción del acceso a datos.
    \item Vista: Generación y presentación del documento presentado al usuario.
    \item Controlador: Redirige las consultas a la vista apropiada.
    \item Plantilla: Generación del HTML a partir de unos datos.
  \end{itemize}  
\end{frame}

\begin{frame}{MVC RoR vs MVC Django}
  \begin{center}
  \begin{tabular}{|ccc|}
    \hline
    RoR&&Django\\
    \hline
    Modelo&=>&Modelo\\
    Vista&=>&Plantilla\\
    Controlador&=>&Vista\\
    Rutas&=>&Controlador\\
    \hline
  \end{tabular}  
  \end{center}
\end{frame}

\subsection{Las piezas}
\begin{frame}{Esquema de consulta}
  \begin{center}
    \includegraphics[width=9cm]{django-esquema.png}
  \end{center}
\end{frame}

\begin{frame}{El controlador}
  \begin{itemize}
    \item Asocia consultas (urls) a vistas.
    \item Usa expresiones regulares para determinar las equivalencias.
    \item Es capaz de capturar parámetros desde la url.
    \item El uso de expresiones regulares nos permite un primer nivel de validación.
  \end{itemize}
\end{frame}

\begin{frame}{Vista}
  \begin{itemize}
    \item Ejecuta las operaciones necesarias para devolver el html al cliente.
    \item Es el encargado de la lógica de programa.
    \item Hace uso del modelo para extraer los datos de la base de datos.
    \item Hace uso de las plantillas para generar el código HTML.
  \end{itemize}
\end{frame}

\begin{frame}{Modelo}
  \begin{itemize}
    \item Define los objetos de nuestra base de datos.
    \item Permite establecer relaciones entre los objetos.
    \item Una vez definidos nos permite crearlos en nuestra base de datos.
    \item Una vez creados nuestros objetos nos permite acceder a ellos cómodamente.
  \end{itemize}
\end{frame}

\begin{frame}{Plantillas}
  \begin{itemize}
    \item Nos permiten crear la presentación de los datos.
    \item Genera html a partir de un código html y una serie de variables y estructuras simples.
    \item Las plantillas que incluye django soportan herencia, bucles, bifurcaciones, variables y filtros.
  \end{itemize}
\end{frame}

\begin{frame}{Middleware}
  \begin{itemize}
    \item Procesa los datos entre varios puntos del sistema.
    \item Nos permite que todas las consultas y respuestas pasen una serie de filtros.
    \item Django integra algunos como el manejo de sesiones o la autenticación.
  \end{itemize}
\end{frame}

\subsection{Extras}
\begin{frame}{Interfaz de administración}
  \begin{itemize}
    \item Es una aplicación que viene integrada con django.
    \item Nos permite administrar nuestros objetos de la base de datos.
    \item Facilita la inserción de datos durante el proceso de desarrollo.
    \item Se puede personalizar, tanto el aspecto como el funcionamiento.
  \end{itemize}
\end{frame}

\begin{frame}{Vistas genéricas}
  \begin{itemize}
    \item Django incorpora algunas vistas genéricas.
    \item Nos permiten hacer las operaciones mas habituales (listar, borrar, crear...).
    \item Agilizan el proceso de desarrollo inicial.
    \item Sirven de base para crear nuestras propias vistas (a partir de ellas).
  \end{itemize}
\end{frame}
