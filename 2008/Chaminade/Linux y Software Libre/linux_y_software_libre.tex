  \section{El Software libre}

  \subsection{Estado actual de la industria del software.}

  \begin{frame}{Estado actual de la industria del software}
    \begin{center}

    \begin{itemize}
      \item Dominio del productor.
      \item Monopolios en nichos de negocio.
      \item Los errores sólo los puede arreglar el productor.
      \item No hay adaptación del software.
    \end{itemize}

    \huge{Copiar y compartir es MALO.}
    \end{center}
  \end{frame}

  \subsection{Claves del software libre}

  \begin{frame}{¿Qué es?}
    \begin{center}
  
    Software Libre es aquel que cumple las cuatro libertades básicas.
    \begin{itemize}
      \item Usarlo dónde, cómo y para lo que quiera.
      \item Redistribuirlo como quiera.
      \item Modificarlo.
      \item Redistribuir las modificaciones.
    \end{itemize}

    Son libertades, en ningún caso obligaciones.

    Es imprescindible disponer del código fuente.
    \end{center}
  \end{frame}

  \begin{frame}{¿Qué no es?}
    \begin{center}
  
    Hay varios conceptos que se confunde con Software Libre.
    \begin{itemize}
      \item Software gratis.
      \item Software abierto.
    \end{itemize}

    Ambas son características que pueden estar en el Software Libre, pero que
    por sí solas no implican que el software sea libre.
    \end{center}
  \end{frame}

  \begin{frame}{¿Por qué es esto?}
    \begin{center}
  
    Motivos Éticos: debería ser así.
    \begin{itemize}
      \item Free Software Foundation.
      \item Ideas formuladas por Richard Stallman.
    \end{itemize}

    Motivos Prácticos: funciona mejor así.
    \begin{itemize}
      \item Open Source Iniciative.
      \item Nuevas posiblidades de negocio (desafío a monopolios).
      \item Ideas formuladas por Eric Raymond (La catedral y el bazar).
    \end{itemize}
    \end{center}
  \end{frame}
  
  \subsection{Un poco de historia}
  
  \begin{frame}{Los comienzos}
    \begin{center}
    \begin{itemize}
      \item 1984: Richard Stallman abandona el AI Lab del MIT para comenzar con el proyecto GNU.
      \item 1985: Richard Stallman funda la Free Software Foundation.
      \item 1990: El proyecto GNU estaba casi completo a falta del núcleo del sistema operativo.
      \item 1991: Aparece en escena el kernel de Linux.
      \item 1994: Aparece la primera version estable oficial de Linux (1.0).
    \end{itemize}
    \end{center}
  \end{frame}

  \begin{frame}{Licencias libres}
    \begin{center}
    \begin{itemize}
      \item Herramientas legales que nos permiten imponer ciertos requisitos a los usuarios y los redistribuidores.
      \item Hacen que se cumplan las cuatro libertades.
      \item Basadas en leyes de propiedad intelectual.
    \end{itemize}

    Existen muchas, las dos principales son:
    \begin{itemize}
      \item GPL: (General Public License) De la Free Software Foundation.
      \item LGPL: (Lesser General Public License) De la Free Software Foundation.
      \item BSD: (Berkeley Software Distribution) De la universidad de Berkeley.
    \end{itemize}

    \end{center}
  \end{frame}

  \subsection{Desarrolladores y usuarios}

  \begin{frame}{¿Quién lo usa y lo desarrolla?}
    \begin{center}
    Hoy por hoy lo usan la mayoría de las grandes empresas del sector informático.

    Lo desarrollan empresas como:
    \begin{itemize}
      \item IBM
      \item Oracle
      \item Sun Microsistem
      \item Novell
      \item Canonical
      \item \dots{}
    \end{itemize}

    \huge{El Software Libre no son 4 Hippies.}
    \end{center}
  \end{frame}
  
  \begin{frame}{Motivaciones del desarrollador de Software Libre}
    \begin{center}
    \begin{itemize}
      \item Motivaciones personales:
      \begin{itemize}
        \item Deseo de aprender.
        \item Deseo de compartir. 
        \item Participar en una comunidad.
      \end{itemize}
      \item Formas de obtener beneficios:
      \begin{itemize}
        \item Obtener reputación.
        \item Motivaciones económicas:
        \begin{itemize}
          \item Creación.
          \item Redistribución.
          \item Integración.
          \item Modificación.
          \item Adaptación.
          \item Soporte.
        \end{itemize}
      \end{itemize}
    \end{itemize}
    \end{center}
  \end{frame}
  
  \begin{frame}{Motivaciones del usuario de Software Libre}
    \begin{center}
    \begin{itemize}
      \item Acceso a la tecnología punta más fácilmente.
      \item Adaptabilidad.
      \item Transparencia.
      \item Problema del cliente cautivo.
      \item Mejor competencia.
      \item Variedad de alternativas.
      \item Ventajas del Software Libre (Más adelante).
      \item Poder ayudar a los proyectos (Más adelante).
    \end{itemize}
    \end{center}
  \end{frame}

  \begin{frame}{Ventajas del Software Libre}
    \begin{center}
  
    El Software Libre consta de ciertas ventajas frente a software 
    \begin{itemize}
      \item Menor coste (aportaciones externas).
      \item Menor coste de distribucion.
      \item Desarrollo más cercano al usuario.
      \item Mantenimiento y soporte independiente del productor.
      \item Adaptabilidad del software.
      \item Mayor facilicad de penetración en nichos de negocio.
    \end{itemize}

    Además combina dos poderosos mecanismos
    \begin{itemize}
      \item Competencia.
      \item Cooperación.
    \end{itemize}
    \end{center}
  \end{frame}

  \begin{frame}{Formas de ayudar}
    \begin{center}
    \begin{itemize}
      \item Código.
      \item Documentación.
      \item Notificando errores y sugerencias.
      \item Usándolo.
      \item Redistribuyendo copias.
      \item Hablando.
    \end{itemize}
    \end{center}
  \end{frame}

  \begin{frame}{Principales obstáculos del Software Libre}
    \begin{center}
    \begin{itemize}
      \item Desconocimiento.
      \item Simplemente se ve como una ''alternativa técnica''.
      \item Técnicas de miedo, desconocimiento y duda.
      \item Problemas legales (Patentes del software).
    \end{itemize}
    \end{center}
  \end{frame}
  
  \begin{frame}{Conclusiones}
    \begin{center}
    \begin{itemize}
      \item Aún es temprano.
      \item El Software Libre demuestra que es viable.
      \item Nuevos modelos de negocio.
      \item Necesario información buena y actual.
      \item Mucha innovación.
      \item El futuro depende de nosotros(profesionales, clientes, empresarios,...)
    \end{itemize}
    \end{center}
  \end{frame}

  \begin{frame}{Conclusiones}
    \begin{center}
    \huge{Éste es uno de esos raros momentos en los que toda una industria puede estar cambiando de paradigma.}
    \end{center}
  \end{frame}
